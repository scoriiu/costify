Introducere în Costify

Costify este sistemul financiar de operare pentru afacerile din România. Acest ghid vă va ajuta să înțelegeți conceptele de bază și să începeți rapid.

Ce este Costify?

Costify este o platformă financiară multi-companie care deservește două audiențe din aceleași date: contabilul care procesează documentele financiare și antreprenorul care trebuie să înțeleagă și să controleze afacerea.

Spre deosebire de software-ul tradițional de contabilitate (Saga, SmartBill) care se concentrează doar pe conformitate, Costify adaugă un strat de inteligență managerială: panouri de control în timp real, bugete, prognoze, clasificarea costurilor și managementul trezoreriei.

Dacă migrați de la Saga, consultați Ghidul de Migrare Saga pentru un tutorial pas cu pas privind importul planului de conturi și a datelor istorice.

Exemplu Rapid

Iată cum puteți conecta o sursă bancară și declanșa prima sincronizare folosind API-ul:

typescript
// Creează o sursă bancară pentru FGO.ro
const source = await costify.sources.create({
  name: 'BT Business via FGO',
  type: 'fgo',
  bankAccountId: account.id,
  config: {
    fgoAccountId: 'acc_12345',
  },
  syncFrequency: 'hourly',
})

// Declanșează sincronizarea inițială
const syncResult = await costify.sources.sync(source.id)
console.log(syncResult.transactionsImported) // 247

Sfat: Fiecare cont bancar poate avea exact o singură sursă activă (FGO sau bancă directă, niciodată ambele) pentru a preveni tranzacțiile duplicate.

Multi-Companie

Costify este construit pentru contabili care administrează sute sau mii de companii client. Fiecare companie este un tenant complet izolat cu propriile date, utilizatori și configurări.

typescript
// Schimbă contextul companiei
const client = costify.tenant('tenant_abc123')

// Toate apelurile ulterioare sunt limitate la această companie
const transactions = await client.transactions.list({
  dateFrom: '2026-01-01',
  status: 'unclassified',
  limit: 50,
})

Securitate: Row-Level Security (RLS) este aplicat la nivel de bază de date. Chiar dacă codul aplicației are un bug, baza de date nu va returna niciodată date aparținând altei companii.
